第1章 · 01 ERP 概念

什么是 ERP？

Enterprise Resource Planning — 企业资源计划。它不是一个软件，而是一套将企业所有业务环节（采购、销售、库存、生产、财务、HR）集成在一个统一平台上的管理思想与信息系统。

数据集成

各部门共享同一套主数据，消除信息孤岛。销售下单 → 自动触发采购/生产 → 实时反映到财务。

流程标准化

将企业业务流程固化到系统中，减少人为差错，确保每笔业务有据可查、有迹可循。

核心价值. 一个订单，从录入到收款，全程在同一个系统中流转，无需重复录入，数据实时同步。

金蝶云星空采用 B/S（Browser/Server） 架构，用户只需浏览器即可登录使用，无需安装任何客户端软件。

第1章 · 05 基础资料

组织架构与基础资料

云星空是多组织架构系统，一个数据中心可以管理多个法人组织和业务组织。

组织架构层级

集团 → 公司（法人组织）→ 部门 → 岗位 → 员工。每个组织有独立的核算体系和业务范围。

基础资料

物料、客户、供应商、科目、币别、计量单位等是系统的"字典"，是所有业务流程的基础数据来源。

分配策略. 基础资料创建后需"分配"到各使用组织，一个集团创建，各子公司共享使用。

第2章 · 01 总账管理

总账：财务核算的核心

总账模块是财务系统的枢纽，接收各业务模块生成的凭证，完成记账、结转和结账操作。

凭证录入与审核

手工录入记账凭证（借/贷方科目+金额），或由业务模块（应收/应付/存货/资产等）自动生成凭证。凭证需经过审核后才能记账。

期末处理

结转损益 → 期末调汇 → 结账。每月必须按时完成，结账后该期间数据锁定不可修改。

账簿查询

总分类账、明细分类账、科目余额表、核算维度余额表，支持上查下钻（凭证→账簿→报表）。

第2章 · 03 应收应付

应收与应付管理

应收、应付模块管理企业与客户、供应商之间的往来款项，是连接业务与财务的桥梁。

应收管理销售出库 → 应收单 → 收款单 → 核销。支持预收款、应收冲应付、坏账处理。账龄分析表监控回款情况。
应付管理采购入库 → 应付单 → 付款单 → 核销。支持预付款、应付冲应收。自动生成应付凭证传递到总账。
账龄分析按客户/供应商维度显示未收/未付金额的时间分布（0-30天、31-60天、60-90天、90天以上）。

第2章 · 06 智能会计平台

多核算体系与凭证自动化

云星空的核心差异化优势：同一笔业务数据，可按不同会计准则自动生成多套凭证和报表。

多核算体系

一套业务数据同时满足法人核算（对外的财务会计）和利润中心核算（对内的管理会计），并行不悖。

凭证模板

配置业务单据到会计凭证的自动映射规则：按物料组/部门/客户等维度自动匹配科目，业务发生即生成凭证。

多账簿

同一套数据可按中国大陆准则、香港准则、IFRS 等生成多套账簿，满足跨境合规需求。

销售管理覆盖从报价到收款的全过程，核心单据流转如下：

第3章 · 04 组织间结算

多组织内部交易处理

当集团公司内不同法人组织之间有采购、销售或跨组织调拨时，需要组织间结算来处理内部交易。

结算场景

组织A向组织B采购物料、跨组织调拨、跨组织领料消耗、集中采购集中结算等。

加价规则

可设置固定加价或百分比加价率，组织间交易自动生成双方的应收应付单据，计入各组织的独立账套。

结算清单

期末生成组织间结算清单，汇总全部内部交易，自动创建内部应收应付单并传递至总账。

第3章 · 05 信用管理

客户信用风险控制

信用额度为每个客户设置信用额度（可区分组织），超出额度时系统自动控制后续交易行为。
控制时点销售订单保存 / 审核、发货通知、销售出库等环节均可设置信用检查点。
超额处理选项：禁止交易 / 预警提示 / 需特批（走审批流）/ 预收款冲抵。
信用计算信用占用 = 已出库未收款 + 已发货未出库 + 已下单未发货。可设置账期维度（如超过60天未收款自动停用信用）。

第4章 · 01 工程数据

BOM + 工艺路线

工程数据是生产制造的"配方"和"步骤说明"：

BOM（物料清单）

定义产品由哪些物料组成，层级结构：成品 → 半成品 → 原材料。支持替代料、可配置BOM。

工艺路线

定义产品加工的顺序步骤：工序1 → 工序2 → …… → 完工。每道工序指定工作中心、标准工时。

数据源头. BOM 和工艺路线是 MRP 运算、生产发料、成本核算的基础，数据准确性直接影响后续所有环节。

第4章 · 02 计划管理

MRP / MPS 运算逻辑

MPS（主生产计划）

对关键成品/半成品做计划，考虑销售预测、销售订单、库存，输出生产计划和采购建议。

MRP（物料需求计划）

基于 BOM 层层展开，计算所有物料的需求量和需求时间：要什么？要多少？什么时候要？什么时候订？

运算逻辑

毛需求 − 现有库存 − 在途量 + 安全库存 = 净需求 → 生成生产计划订单 / 采购申请。

工序流转与进度跟踪

工序计划

生产订单审核后按工艺路线生成工序计划，明确每道工序的顺序、工作中心、计划工时。

工序汇报

工人完成工序后报工：合格数量、废品数量、实际工时。系统自动计算完工进度和在制品数量。

工序转移

支持顺序流转（完成A工序才能开始B）和并行流转（多工序同时加工），灵活适应不同生产模式。

第4章 · 06 成本管理

生产成本核算

产品成本 = 材料成本 + 人工成本 + 制造费用，云星空支持多种核算方法：

品种法

按产品品种归集成本，适合大批量单步骤生产

分批法

按生产批次/订单归集成本，适合小批量多品种生产

分步法

按生产步骤逐步结转，适合多工序连续加工

费用分配

制造费用可按工时/产量/机时等标准分配到各产品

一笔订单的全流程之旅

看一笔销售业务如何贯穿企业五大模块：

① 客户在电商平台下单 → 全渠道营销 接收订单同步至云星空

② 系统进行信用检查 → 自动生成 销售订单（SD）

③ MRP 运算发现成品库存不足 → 生成 生产计划订单（PP）

④ 生产领料、工序加工、完工入库 → 生产+车间+成本 协同

⑤ 部分物料库存不足 → 自动生成 采购申请（MM）→ 采购入库

⑥ 仓库发货出库 → 生成 应收单 → 客户付款 → 出纳 记录到账

⑦ 全部单据自动生成凭证 → 总账 记账 → 报表 实时反映
