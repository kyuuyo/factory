stateDiagram-v2

SUMMARY

数据模型设计要点

1-N-N 单据结构

WareHouseOrder(主) → Detail(物料行) → Receive(批次行)，层级清晰，每层独立 Trace。

双状态机驱动

ProductionStatus(库存0-8) + InOutStaus(单据) + FHTZStatus，三状态机协同推进。

IAuditable 全覆盖

所有实体统一实现 CreateTime/Creator/ModifyTime/Modifier/IsDelete，审计追踪零遗漏。

Trace 快照模式

WareHouseStockTrace + LabelTrace 每次操作克隆快照，记录时刻全量状态 + 关联单据。

ERP 解耦设计

M_WareHouseOrder 中间表缓存 ERP 数据，MOM 本地存取，按需同步，降低 ERP 依赖。

统一请求响应

SearchCriteriaModel 搜索 + BaseResponseModel 响应 + DataPageResponseModel 分页，全 API 统一契约。
