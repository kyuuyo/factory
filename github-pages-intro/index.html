01 · OVERVIEW

什么是
GitHub Pages？

GitHub Pages 是 GitHub 提供的免费静态网站托管服务。你把 HTML/CSS/JS 文件推送到仓库，它自动变成可访问的网站。

🎯 核心定位

托管静态内容：HTML、CSS、JS、图片、前端构建产物

❌ 不能做的事

不支持后端运行（无 PHP/Node/Python 运行时），无数据库

🔗 访问方式

用户名.github.io/仓库名

02 · THREE TYPES

三种 GitHub Pages 站点

用户站点仓库名必须是 用户名.github.io，访问地址 https://用户名.github.io。每个 GitHub 账号只能有一个。
组织站点与用户站点类似，仓库名 组织名.github.io。用于组织/公司品牌主页。
项目站点任意仓库都可启用，访问地址 https://用户名.github.io/仓库名。无数量限制。
← 我们的 factory 仓库用的就是这种
我们的站点： https://kyuuyo.github.io/factory — 来自 kyuuyo/factory 仓库的项目站点

04 · CONFIGURATION

两种配置方式

在仓库的 Settings → Pages 里选择发布源：

方式一：Branch 部署

指定分支 + 目录 → 推送即发布。适用于纯静态文件或本地已构建好的项目。

master / docs main / root

方式二：GitHub Actions

通过 CI 工作流自动构建。适用于 React/Vue/Hugo 等需要编译后才能部署的前端项目。

jekyll hugo next.js static export

05 · DOMAIN & HTTPS

自定义域名
& HTTPS

默认提供 .github.io 子域名，也支持绑定你自己的域名。

🔒 自动 HTTPS

Let's Encrypt 自动签发 SSL 证书，无需任何配置，HTTPS 默认强制开启。

🌐 自定义域名

Settings → Pages → Custom domain 填入域名，再在 DNS 添加 CNAME 记录指向 用户名.github.io。

📄 CNAME 文件

在仓库根目录放一个 CNAME 文件，内容为你绑定的域名。GitHub 会自动识别。

06 · JEKYLL INTEGRATION

内置 Jekyll 支持

GitHub Pages 原生支持 Jekyll（静态网站生成器），推送 Markdown 自动转为静态页面。

  • 直接写 Markdown — 推送 .md 文件，自动渲染为 HTML 页面
  • 主题市场 — Settings → Pages → Theme Chooser 选择免费主题
  • Front Matter — YAML 定义页面标题、布局、参数
  • Liquid 模板 — 支持变量、循环、条件等模板语法
  • _config.yml — 站点级的全局配置
跳过 Jekyll. 在根目录放一个空文件 .nojekyll，GitHub 就会当做纯静态文件托管——我们的 factory 仓库就用了这种方式。

07 · LIMITS

限制与额度

仓库大小推荐 1 GB 以内（含所有分支与历史）
站点大小发布后的站点不能超过 1 GB
流量带宽每月 100 GB 带宽（软限制，极少触发）
构建次数每小时最多 10 次构建（纯静态文件不受限）
站点数量无限制 —— 每个仓库都可以有一个 Pages 站点
自定义域名免费支持，无数量限制
实际使用. 对于文档站、个人主页、技术演示、博客等场景，这些限制几乎永远不会触碰。GitHub Pages 本质上是鼓励使用的。

09 · COMPARISON

静态托管服务对比

GitHub Pages免费 · 与 GitHub 深度集成 · 全球 CDN · 自动 HTTPS · 限制 1GB
Vercel免费 100GB 带宽 · 自动构建 · Serverless · 适合前端框架
Netlify免费 100GB · 表单功能 · 分支预览 · A/B 测试
Cloudflare Pages无限带宽 · 全球 330+ 节点 · 免费 Workers · 最快
GitLab Pages免费 · CI/CD 集成 · 和 GitHub Pages 类似
选 GitHub Pages 的理由：代码已经在 GitHub 上了，零额外配置，自动 HTTPS，推送即上线。对纯静态演示项目来说最省事。
